# Building rusb for WASI

This document explains how to cross-compile `rusb` for WASI, linking against the guest-side `libusb-wasi` implementation.

## Architecture

```
Rust code → rusb → libusb-wasi (guest) → WASI-USB interface → host implementation → syscalls
```

`rusb` depends on `libusb1-sys`, which uses `pkg-config` to locate `libusb`. For WASI cross-compilation, we redirect `pkg-config` to find the WASI-compiled `libusb-wasi.a` (from `../libusb-wasi/`) instead of the system library.

## Prerequisites

- **Rust** with `wasm32-wasip2` target: `rustup target add wasm32-wasip2`
- **libusb-wasi**: A pre-built `libusb-wasi.a` (see `../libusb-wasi/BUILDING_WASI.md`)

## Setting up the Cross-Compile Sysroot

Since Rust's `pkg-config` build helper doesn't support `$CHOST`-prefixed `pkg-config`, we configure it via environment variables pointing to a sysroot containing `libusb-wasi`. See [rusb cross-compilation docs](https://github.com/a1ien/rusb#cross-compiling).

### 1. Create the sysroot structure

```sh
mkdir -p wasi-sysroot/usr/lib/pkgconfig
mkdir -p wasi-sysroot/usr/include/libusb-1.0
```

### 2. Link the library, header, and component type

```sh
ln -s /path/to/libusb-wasi/libusb-wasi.a  wasi-sysroot/usr/lib/libusb-1.0.a
ln -s /path/to/libusb-wasi/libusb/libusb.h wasi-sysroot/usr/include/libusb-1.0/libusb.h
cp /path/to/wasi-usb/usb-wasi-cguest/bindings/cguest_component_type.o wasi-sysroot/usr/lib/
```

> **Note**: `cguest_component_type.o` contains the `component-type:cguest` custom section describing the WASI-USB WIT world. This is generated by `wit-bindgen` and is essential for `wasm-component-ld` to resolve custom imports.

### 3. Create `libusb-1.0.pc`

Create `wasi-sysroot/usr/lib/pkgconfig/libusb-1.0.pc`:

```ini
prefix=/usr
exec_prefix=${prefix}
libdir=${exec_prefix}/lib
includedir=${prefix}/include

Name: libusb-1.0
Description: WASI libusb (cross-compiled)
Version: 1.0.28
Libs: -L${libdir} -lusb-1.0
Cflags: -I${includedir}/libusb-1.0
```

> Paths are sysroot-relative. `PKG_CONFIG_SYSROOT_DIR` will be prepended automatically.

## Project Configuration

Your workload project needs:

**`Cargo.toml`** — configured as a reactor:

```toml
[package]
build = "build.rs"

[lib]
crate-type = ["cdylib"]

[dependencies]
rusb = { path = "../.." }  # Points to rusb-wasi root
```

**`build.rs`** — links the component-type object:

```rust
use std::path::PathBuf;

fn main() {
    let sysroot = PathBuf::from(env!("CARGO_MANIFEST_DIR"))
        .join("wasi-sysroot/usr/lib");
    println!("cargo:rustc-link-arg={}",
        sysroot.join("cguest_component_type.o").display());
}
```

> This links the `component-type:cguest` custom section into the final binary, allowing `wasm-component-ld` to resolve the custom WASI-USB imports during componentization.

**`src/lib.rs`** — entry point matching `cguest` bindings:

```rust
#[unsafe(no_mangle)]
pub extern "C" fn exports_wasi_cli_run_run() -> bool {
    // Your workload logic using rusb
    true
}
```

## Cross-Compilation

Set the `pkg-config` environment variables as described in the [rusb cross-compilation guide](https://github.com/a1ien/rusb#cross-compiling):

```sh
SYSROOT=/path/to/wasi-sysroot

export PKG_CONFIG_DIR=
export PKG_CONFIG_LIBDIR=${SYSROOT}/usr/lib/pkgconfig:${SYSROOT}/usr/share/pkgconfig
export PKG_CONFIG_SYSROOT_DIR=${SYSROOT}
export PKG_CONFIG_ALLOW_CROSS=1
export LIBUSB_STATIC=1

cargo build --target wasm32-wasip2 --release
```

This directly produces a WASI component at `target/wasm32-wasip2/release/<name>.wasm`. No additional `wasm-tools` steps are needed — `wasm-component-ld` handles the componentization automatically using the `component-type:cguest` metadata from `cguest_component_type.o`.

## Verification

```sh
file target/wasm32-wasip2/release/<name>.wasm
# Expected: WebAssembly (wasm) binary module version 0x1000d
```

## Quick Start

A ready-to-use example is provided in `examples/wasi-workload/`:

```sh
cd examples/wasi-workload
chmod +x build.sh
./build.sh
```
